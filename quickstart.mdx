## Get ImageFlow running in 3 steps

Choose your preferred deployment method and have ImageFlow running in minutes.

<Tabs>
  <Tab title="Docker (Recommended)">
    The fastest way to get ImageFlow running is with Docker.

    ### Prerequisites
    - Docker and Docker Compose installed
    - 2GB free RAM
    - 10GB free disk space

    ### Step 1: Clone the repository
    ```bash
    git clone https://github.com/username/ImageFlow.git
    cd ImageFlow
    ```

    ### Step 2: Configure environment
    ```bash
    # Copy example environment file
    cp .env.example .env

    # Edit the configuration (required)
    nano .env
    ```

    Set these required variables:
    ```env
    API_KEY=your-secret-api-key-here
    STORAGE_TYPE=local
    REDIS_ENABLED=true
    ```

    ### Step 3: Start the service
    ```bash
    # Start with Redis (recommended)
    docker-compose up -d

    # Or start without Redis
    docker-compose -f docker-compose-no-redis.yaml up -d
    ```

    ImageFlow will be available at `http://localhost:8080`

    <Tip>The first startup may take a few minutes to build and download dependencies.</Tip>
  </Tab>

  <Tab title="Local Development">
    Run ImageFlow locally for development or customization.

    ### Prerequisites
    - Go 1.23+ ([Download](https://golang.org/dl/))
    - Node.js 18+ ([Download](https://nodejs.org/))
    - libvips development libraries
    - Redis (optional but recommended)

    ### Step 1: Install libvips
    <CodeGroup>
      ```bash Ubuntu/Debian
      sudo apt update
      sudo apt install libvips-dev
      ```
      
      ```bash macOS
      brew install vips
      ```
      
      ```bash CentOS/RHEL
      sudo yum install vips-devel
      ```
    </CodeGroup>

    ### Step 2: Clone and configure
    ```bash
    git clone https://github.com/username/ImageFlow.git
    cd ImageFlow

    # Install Go dependencies
    go mod tidy

    # Configure environment
    cp .env.example .env
    nano .env
    ```

    ### Step 3: Build frontend (optional)
    ```bash
    cd frontend
    npm install
    npm run build
    cd ..
    ```

    ### Step 4: Start the service
    ```bash
    # Start the backend
    go run main.go
    ```

    ImageFlow will be available at `http://localhost:8080`
  </Tab>
</Tabs>

## Verify Installation

Once ImageFlow is running, verify everything works:

### 1. Check the web interface
Visit `http://localhost:8080` in your browser. You should see the ImageFlow upload interface.

### 2. Test API connectivity
```bash
curl http://localhost:8080/api/config
```

### 3. Upload your first image
1. Click the upload area or drag an image
2. Watch it process automatically
3. View the optimized versions in different formats

## Common Issues

<AccordionGroup>
  <Accordion title="Port 8080 already in use">
    Change the port in your `.env` file:
    ```env
    PORT=8081
    ```
  </Accordion>
  
  <Accordion title="libvips not found">
    Make sure libvips development libraries are installed:
    ```bash
    # Ubuntu/Debian
    sudo apt install libvips-dev
    
    # macOS
    brew install vips
    ```
  </Accordion>
  
  <Accordion title="Permission denied errors">
    Ensure the `static/images` directory is writable:
    ```bash
    mkdir -p static/images
    chmod 755 static/images
    ```
  </Accordion>
</AccordionGroup>
